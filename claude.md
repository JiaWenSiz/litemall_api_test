


# 项目说明：Jenkins CI/CD 持续集成与交付平台 

## 一、项目概述 

本项目基于 Jenkins 搭建自动化持续集成、持续交付体系，实现代码自动构建、定时触发、构建结果通知、构建日志分析等能力。 Jenkins 部署在 Docker 容器中，用于自动化构建、接口测试、任务调度与结果告警。 

## 二、运行环境

- 部署方式：Docker 容器 - Jenkins 
- 容器 ID：d3f795c63abb - Jenkins 
- 容器名称：jenkins - Jenkins 
- 镜像：jenkins/jenkins:lts 
- 映射端口：8080、50000 

## 三、核心配置 

### 1. 定时构建规则

- Cron 表达式：H * * * *

- 执行频率：每小时执行一次（自动分散负载，避免整点并发压力） 

### 2. 通知方式 

- 优先使用：邮件通知（SMTP） 

- 备选：飞书机器人 / 企业微信机器人 Webhook 通知 

- 通知场景：构建成功、构建失败、构建不稳定 

### 3. 构建流程 

1. 代码拉取 

2. 项目编译 / 构建 

3. 自动化接口测试（Litemall 相关接口测试） 

4. 构建结果判定 

5. 邮件/飞书通知发送 

6. 构建历史留存与分析 

## 四、构建状态说明 

- Success（蓝色）：构建成功 

- Failure（红色）：构建失败 

- Unstable（黄色）：构建完成但存在测试失败/警告 

## 五、常用操作 

- 进入容器：docker exec -it jenkins /bin/bash 

- 查看容器：docker ps | grep jenkins 

- 查看构建日志：Jenkins 项目 → 构建编号 → 控制台输出
